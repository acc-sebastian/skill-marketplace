## Stage by stage

| Stage | Trigger | What runs | Gate |
|-------|---------|-----------|------|
| **Contribute** | web form (`new-skill.yml`) or PR | `scaffold-skill.yml` turns a form submission into a PR | ‚Äî |
| **Validate** | every PR + push | `validate_skills.py` ‚Äî JSON-Schema, naming, unique IDs, links, `SKILL.md` lint | required check `validate` |
| **Quality** | every PR + push | `scan_skills.py` ‚Äî secrets fail the build; PII/injection warn | part of `validate` |
| **Review** | PR | CODEOWNER approval | branch protection on `main` |
| **Build / Deploy** | merge to `main` | `build_site.py` ‚Üí site + `catalog.json` + schema ‚Üí `build-site.yml` deploys to Pages | ‚Äî |
| **Distribute** | merge to `main` | `release.yml` publishes a GitHub Release `id@version` with assets; the repo is a Claude Code plugin marketplace; `skill_cli.py` installs from the catalog | ‚Äî |
| **Notify** | skill change on `main` | `notify-teams.yml` posts a Teams card (skips without webhook) | ‚Äî |
| **Maintain** | schedules | `stale-check.yml` (review-due issues), `smoke-test.yml` (LLM/model-drift), `auto-archive.yml` (retire past sunset), `insights.yml` (priority report) | ‚Äî |

## Single sources of truth

- **Skills:** `plugins/accilium-skills/skills/<id>/{SKILL.md, metadata.json}` on `main`.
- **Schema:** `schema/skill.schema.json` (validated in CI, published to the site).
- **Site & catalog:** generated by `scripts/build_site.py` ‚Äî never edit `docs/index.html`/`docs/catalog.json` by hand.
