# The Skill Marketplace Pipeline (end to end)

Every stage from a skill idea to it being live, distributed, and maintained.
All GitHub-native. This is the "verdrahtete" full pipeline (Roadmap Phase 5).

```
 CONTRIBUTE        VALIDATE            QUALITY            REVIEW      BUILD/DEPLOY        DISTRIBUTE           NOTIFY            MAINTAIN
 ──────────        ────────            ───────            ──────      ────────────        ──────────           ──────            ────────
 web form ─┐       schema check        secret scan        CODEOWNER   build_site.py       plugin marketplace   Teams card        stale-check (cron)
 (issue)   ├─ PR → naming/uniqueness → PII/injection  →   review   →  → site+catalog  →   + releases (id@ver)  on skill change   smoke-test / drift (cron)
 PR direct ┘       link check          (validate.yml)     +green      deploy to Pages     + CLI install                          auto-archive (cron)
                   (validate.yml)                         checks                                               feedback links    insights report (cron)
```

## Stage by stage

| Stage | Trigger | What runs | Gate |
|-------|---------|-----------|------|
| **Contribute** | web form (`new-skill.yml`) or PR | `scaffold-skill.yml` turns a form submission into a PR | — |
| **Validate** | every PR + push | `validate_skills.py` — JSON-Schema, naming, unique IDs, links, `SKILL.md` lint | required check `validate` |
| **Quality** | every PR + push | `scan_skills.py` — secrets fail the build; PII/injection warn | part of `validate` |
| **Review** | PR | CODEOWNER approval | branch protection on `main` |
| **Build / Deploy** | merge to `main` | `build_site.py` → site + `catalog.json` + schema → `build-site.yml` deploys to Pages | — |
| **Distribute** | merge to `main` | `release.yml` publishes a GitHub Release `id@version` with assets; the repo is a Claude Code plugin marketplace; `skill_cli.py` installs from the catalog | — |
| **Notify** | skill change on `main` | `notify-teams.yml` posts a Teams card (skips without webhook) | — |
| **Maintain** | schedules | `stale-check.yml` (review-due issues), `smoke-test.yml` (LLM/model-drift), `auto-archive.yml` (retire past sunset), `insights.yml` (priority report) | — |

## Single sources of truth

- **Skills:** `plugins/sbo-skills/skills/<id>/{SKILL.md, metadata.json}` on `main`.
- **Schema:** `schema/skill.schema.json` (validated in CI, published to the site).
- **Site & catalog:** generated by `scripts/build_site.py` — never edit `docs/index.html`/`docs/catalog.json` by hand.

## Secrets (optional — features skip cleanly without them)

| Secret | Enables |
|--------|---------|
| `ANTHROPIC_API_KEY` | LLM smoke-test / model-drift check |
| `TEAMS_WEBHOOK_URL` | Teams notifications |

Add under **Settings → Secrets and variables → Actions**.
